HTML Structure

Responsive Grid Sytem

The grid system makes it easy to create well-proportioned layouts. The extended 1200px Skeleton framework, based on the 960.gs framework, splits the grid into sixteen columns. Columns are created by adding simple, intuitive classes to your HTML elements.

The grid elements must be wrapped in a .container element, so you end up with something like this:

<div class="container">
   <div class="five columns">five columns</div>
   <div class="eleven columns">eleven columns</div>
</div>

A full width page is 16 columns, while a page with a sidebar is split into 11 and 4 columns, with a 1 column offset

Columns on a Full Width Page (16 Columns)
1/2 Columns (2 cols)
<div class="container">
   <div class="eight columns">eight columns</div>
   <div class="eight columns">eight columns</div>
</div>
1/4 Columns (4 cols)
<div class="container">
   <div class="four columns">four columns</div>
   <div class="four columns">four columns</div>
   <div class="four columns">four columns</div>
   <div class="four columns">four columns</div>
</div>
1/3 Columns (2 cols)
<div class="container">
   <div class="one-third column">One third column</div>
   <div class="two-thirds column">Two thirds column</div>
</div>
Nested Columns

If you nest columns (use columns within columns), you'll need to use the .alpha and .omega classes on your first and last columns, respectively, to account for the margins.

<div class="container">
   <div class="twelve columns">
      <div class="four columns alpha">four columns</div>
      <div class="eight columns omega">eight columns</div>
   </div>
</div>
Offsets

To leave a column gap between columns, use the .offset-by-X class. Valid values for X are 'one' through 'fifteen'

<div class="container">
   <div class="eleven column">Eleven column</div>
   <div class="four columns offset-by-one">Four column</div>
</div>

 

 

Navigation

The navigation is built with unordered lists and links. The full size menu is CSS-driven.

Responsive: The mobile dropdown is displayed when the viewport is less than 480px wide, to save screen real estate on mobile devices.

<!-- Start Nav Menu -->
<ul class="menu" id="nav">
    <li><a href="#homepage">Home</a></li>
    <li><a href="#about">About</a></li>
    <li><a href="#party">Wedding Party</a></li>
    <li><a href="#location">Location</a></li>
    <li><a href="#registry">Registry</a></li>
    <li><a href="#rsvp">RSVP</a></li>
</ul>
<!-- End Nav Menu -->
<!-- Start Dropmenu for mobile -->        
<select class="dropmenu" name="dropmenu" onChange="moveTo(this.value)">
    <option value="" selected="selected">Menu</option>
    <option value="#homepage">Home</option>
    <option value="#about">About</option>
    <option value="#party">Wedding Party</option>
    <option value="#location">Location</option>
    <option value="#registry">Registry</option>
    <option value="#rsvp">RSVP</option>
</select>
<!-- End Dropmenu for mobile -->

Responsive Images

The Skeleton framework features a couple different tricks for achieving "Responsive Images". What are responsive images? In short, they are images that scale with the layout... so as the layout gets smaller, they scale down, instead of breaking the layout. Just add the class .scale-with-grid to your image tag

<img src="images/gallery/avatar-420x270.jpg" class="scale-with-grid" alt="" />

 

Adding more content sections to the page

It is relatively easy to add more content sections to the site, just copy and paste one of the id sections e.g. #about and then paste it below that section, to get the parallax to work you will need to open the cutom.js file and add a #parallax6 as indicated and so on. You will also need to add another link in the navigation that links to that new id.

Prefixfree - prefix-free lets you use only unprefixed CSS properties everywhere. It works behind the scenes, adding the current browser’s prefix to any CSS code, only when it’s needed. No more -webkit- or -moz-. This won't work locally in Chrome, beeds to be used with a live site (see link for fix to this).

Modernizr - Modernizr is a small JavaScript library that detects the availability of native implementations for next-generation web technologies, i.e. features that stem from the HTML5 and CSS3 specifications.

iOS Timer - iOS6 suffers from a bug that kills timers that are created while a page is scrolling. This library fixes that problem by recreating timers after scrolling finishes (with interval correction).

jQuery - jQuery is a Javascript library that greatly reduces the amount of code that you must write.

Pretty Photo - Pretty Photo is a jQuery lightbox clone. Not only does it support images, it also has support for videos, flash, YouTube, iframes and ajax.

Countdown - jquery script for the countdown.
